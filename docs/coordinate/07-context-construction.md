# 07 上下文构造：Controller 如何为每个技能定制信息环境

Controller 最核心的协调动作不是"调用技能"，而是"构造上下文"。每个技能看到的不是同一个世界——Controller 为每个技能定制了不同的信息环境，使它只看到自己需要的信息，只做自己该做的事。

## 核心命题

```
  ┌──────────────────────────────────────────────────────────┐
  │  协调的本质不是"调度"，而是"上下文构造"。             │
  │                                                          │
  │  同一个 AI，在不同上下文中，表现出完全不同的行为。     │
  │  Controller 不是指挥官——它是信息建筑师。              │
  │                                                          │
  │  上下文构造的七个维度：                                 │
  │  1. 内容选择——包含什么信息                              │
  │  2. 内容排除——不包含什么信息                            │
  │  3. 内容排序——信息的呈现顺序                            │
  │  4. 内容框架——信息的角色和视角                          │
  │  5. 内容隔离——上下文是共享还是隔离                      │
  │  6. 内容变换——信息在技能间如何转换                      │
  │  7. 内容生命周期——上下文何时创建、使用、丢弃            │
  └──────────────────────────────────────────────────────────┘
```

## 三种上下文构造模式

```
  ┌──────────────────────────────────────────────────────────┐
  │  模式 1: 技能注入（主链）                                │
  │  ────────────────────────                                │
  │  Controller 加载技能 → 技能内容追加到对话上下文          │
  │  → 技能共享对话历史                                     │
  │  → 技能能看到之前的所有信息                             │
  │                                                          │
  │  模式 2: 子智能体派发（嵌套）                            │
  │  ────────────────────────                                │
  │  Controller 构造全新的 Prompt → 派发独立子智能体         │
  │  → 子智能体不共享对话历史                               │
  │  → 子智能体只看到 Controller 给它的信息                  │
  │                                                          │
  │  模式 3: 启动注入（会话诞生）                            │
  │  ────────────────────────                                │
  │  插件在第一个用户消息前注入 using-superpowers            │
  │  → 这是 Controller 的"出生证明"                        │
  │  → 定义了 Controller 的身份和行为规则                    │
  │                                                          │
  │  三种模式的上下文隔离度递增：                            │
  │  注入 < 派发 < 启动                                     │
  │  共享度   部分隔离  完全隔离  全局定义                   │
  └──────────────────────────────────────────────────────────┘
```
