# 07 何时保留裁量空间

裁量空间不是越小越好。全程最小 = 零灵活性 = 系统不可用。策略性地保留裁量空间和策略性地压缩它同样重要。

## 弱指令的价值

```
  ┌──────────────────────────────────────────────────────────┐
  │  Superpowers 中约 60% 的指令是 Level 0-1（建议性/指示性）│
  │  → 这些指令故意保留了裁量空间                            │
  │  → 为什么？                                             │
  │                                                          │
  │  场景 1: 用户偏好不应被强制                              │
  │  ──────────────────────────────                          │
  │  "User preferences for spec location                     │
  │   override this default"                                │
  │  → 没有全大写，没有标签，没有否定                       │
  │  → 因为用户偏好应该有裁量空间                           │
  │  → 如果用 <HARD-GATE> → 用户失去了选择权               │
  │                                                          │
  │  场景 2: 防止无限循环                                    │
  │  ────────────────                                        │
  │  "Fix any issues inline. No need to re-review             │
  │   — just fix and move on."                               │
  │  → "just fix and move on" 故意轻松                       │
  │  → 如果用 MANDATORY → AI 可能无限自审                   │
  │  → 保留裁量空间 = 允许 AI 判断"够好了"                 │
  │                                                          │
  │  场景 3: 鼓励提问                                        │
  │  ──────────                                              │
  │  "It's always OK to pause and clarify.                   │
  │   Don't guess or make assumptions."                      │
  │  → "It's always OK" 是许可式，不是命令式                │
  │  → 如果用 MANDATORY ASK QUESTIONS                       │
  │    → AI 可能强迫自己提问（即使不需要），拖慢速度       │
  │  → 许可式降低了提问的心理门槛，但不强制每事必问        │
  └──────────────────────────────────────────────────────────┘
```

## 裁量空间预算

```
  ┌──────────────────────────────────────────────────────────┐
  │  可以把裁量空间想象成"预算"——                            │
  │  AI 在一个会话中能投入的"服从努力"是有限的。            │
  │                                                          │
  │  预算分配原则：                                          │
  │  - 关键流程节点 → 高预算（低裁量空间）                   │
  │  - 具体实现细节 → 低预算（高裁量空间）                   │
  │  - 用户偏好 → 零预算（完全裁量空间）                     │
  │                                                          │
  │  预算分配示意：                                          │
  │  ┌──────────────────────────────────────────────┐       │
  │  │  预算   指令类型             举例             │       │
  │  │  ────   ────────             ────             │       │
  │  │  30%    铁律                 Iron Laws (3条)  │       │
  │  │  25%    否定性规则           Never (20+条)    │       │
  │  │  20%    命令性规则           MUST (10+条)     │       │
  │  │  15%    指示性规则           普通指令 (50+条) │       │
  │  │  10%    建议性规则           可选建议 (5-10条) │       │
  │  └──────────────────────────────────────────────┘       │
  │                                                          │
  │  如果所有指令都是 30% 预算 → 总需求 = 3000% → 严重超支  │
  │  → AI 开始选择性忽略 → 哪些被忽略不可预测               │
  │                                                          │
  │  金字塔分配：                                             │
  │  3×30% + 20×25% + 10×20% + 50×15% + 8×10%              │
  │  = 90% + 500% + 200% + 750% + 80% = 1620%               │
  │  → 仍然超支，但高强度指令占比低                          │
  │  → AI 至少会对最关键的指令保持服从                       │
  └──────────────────────────────────────────────────────────┘
```

## 决策框架：何时压缩，何时保留

```
  ┌──────────────────────────────────────────────────────────┐
  │  压缩裁量空间（用强指令）当：                            │
  │  ──────────────────────                                  │
  │  1. AI 的自然倾向与期望行为相反                          │
  │     → AI 倾向于跳过测试 → 用 Iron Law 压缩              │
  │     → AI 倾向于声称完成 → 用 验证 Iron Law 压缩         │
  │  2. 违反后果严重                                         │
  │     → 跳过 TDD → 未验证代码 → 技术债                    │
  │     → 未验证声称 → 用户信任丧失                         │
  │  3. AI 有系统性的偷懒倾向                                │
  │     → "就这一次"的心态 → 全称量词封堵                   │
  │     → "差不多做了"的心态 → 道德否定封堵                │
  │                                                          │
  │  保留裁量空间（用弱指令）当：                            │
  │  ──────────────────────                                  │
  │  1. AI 的自然倾向与期望行为一致                          │
  │     → AI 自然会检查文件 → 不需要强制                    │
  │     → AI 自然会问问题 → 不需要强制                      │
  │  2. 用户应有选择权                                       │
  │     → 偏好设置 → 保留裁量                               │
  │     → 执行方式 → 保留裁量                               │
  │  3. 过度执行比不执行更糟                                 │
  │     → 无限自审 → 留"move on"出口                       │
  │     → 强迫提问 → 留"OK to pause"许可                   │
  │                                                          │
  │  核心判断：                                              │
  │  AI 的"自然倾向"和"期望行为"一致 → 保留裁量空间       │
  │  AI 的"自然倾向"和"期望行为"相反 → 压缩裁量空间       │
  │  → 裁量空间 = 纠偏力                                    │
  │  → 偏差越大，需要的纠偏力越强                           │
  │  → 没有偏差，不需要纠偏力                                │
  └──────────────────────────────────────────────────────────┘
```

## 裁量空间与指令可信度

```
  ┌──────────────────────────────────────────────────────────┐
  │  保留裁量空间的另一个理由：                               │
  │  如果所有指令都是"不可绕过"的，                        │
  │  AI 无法判断哪些真正重要。                               │
  │                                                          │
  │  类比：交通系统                                          │
  │  - 红灯 = 不可绕过（铁律级）                            │
  │  - 黄灯 = 裁量空间（判断是否停下）                      │
  │  - 绿灯 = 完全裁量（自由通行）                          │
  │  - 路标 = 建议性（建议限速）                            │
  │                                                          │
  │  如果所有信号都是红灯 → 司机无法行驶                    │
  │  如果所有信号都是绿灯 → 司机无法预判危险               │
  │  → 混合信号 = 有效交通系统                              │
  │                                                          │
  │  Superpowers 的信号混合：                                │
  │  - 红灯（3 条 Iron Law）→ 不可绕过                     │
  │  - 黄灯（20+ Never 规则）→ 几乎不可绕过                │
  │  - 绿灯（50+ 指示性规则）→ 应该做但有裁量              │
  │  - 路标（5-10 建议性规则）→ 建议但可忽略               │
  │  → 四种信号混合 = 有效指令系统                          │
  │  → AI 知道什么是"必须做"、什么是"应该做"、            │
  │    什么是"可以做"                                       │
  │  → 梯度差 = 优先级信号 = 有效的裁量空间分配             │
  └──────────────────────────────────────────────────────────┘
```
