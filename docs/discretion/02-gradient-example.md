# 02 裁量空间梯度示例

同一个意图——"先写测试再写代码"——用不同表达，裁量空间从 90% 递减到 1%。

## 六级梯度

### Level 0: "你可以在写代码前考虑写测试"

```
  裁量空间：~90%

  AI 可以：
  ✓ 决定"这次不需要测试"
  ✓ 决定"先写代码，之后考虑测试"
  ✓ 决定"用户只是建议，不是要求"
  ✓ 完全忽略这条建议
  ✓ 决定"考虑了，但觉得没必要"

  → "考虑"赋予了 AI 不做的权利
  → "可以"进一步确认了权利
  → 裁量空间接近最大值
```

### Level 1: "写代码前先写测试"

```
  裁量空间：~60%

  AI 可以：
  ✓ 理解为"总体上应该先测试"（但这次例外）
  ✓ 理解为"先写一些探索性代码，然后从测试开始"
  ✓ 理解为"对重要功能先测试，简单功能不必"
  ✗ 不能完全忽略（没有"可以"或"建议"）

  → 去掉了"考虑"和"可以"——建议变成指示
  → 但"先"的含义模糊——AI 可以重新解读"先"
  → 裁量空间显著缩小，但仍有出口
```

### Level 2: "你必须先写测试"

```
  裁量空间：~30%

  AI 可以：
  ✓ 在极少数情况下声称"这次情况特殊"
  ✓ 理解为"必须先写至少一个测试"（但不用 TDD 完整流程）
  ✗ 不能忽略（"必须"是义务）
  ✗ 不能重新解读顺序（"先"被"必须"强化了）

  → "必须"消灭了"可选"解读
  → 但 AI 可能找到"特殊情况"出口
  → 裁量空间缩小到约三分之一
```

### Level 3: "绝不先写代码再写测试"

```
  裁量空间：~10%

  AI 可以：
  ✓ 在极极端情况下合理化（非常困难）
  ✓ 重新定义"什么是代码"（"探索性代码不算"）
  ✗ 不能声称"这次例外"（"绝不"消灭了例外）
  ✗ 不能重新解读顺序

  → "绝不"消灭了"例外"出口
  → 只剩下"重新定义"这条路径
  → 裁量空间缩小到约十分之一
```

### Level 4: "NO PRODUCTION CODE WITHOUT A FAILING TEST FIRST"

```
  裁量空间：~3%

  AI 可以：
  ✓ 几乎没有路径可以绕过
  ✗ 不能忽略（全大写 + 铁律句式 + 代码块）
  ✗ 不能声称例外（NO 消灭了例外）
  ✗ 不能重新定义（PRODUCTION CODE 精确定义了范围）
  ✗ 不能后补（FIRST 锁定了时序）

  → 唯一剩余的裁量：什么算"失败测试"的定义
  → 裁量空间极小
```

### Level 5: \<HARD-GATE\> + 全称量词 + 反预期

```
  <HARD-GATE>
  Do NOT write any code until design is approved
  This applies to EVERY project
  regardless of perceived simplicity.
  </HARD-GATE>

  裁量空间：~1%

  AI 可以：
  ✓ 几乎没有路径
  ✗ 标签封堵了"降级"路径
  ✗ Do NOT 封堵了"忽略"路径
  ✗ EVERY 封堵了"例外"路径
  ✗ "regardless of perceived simplicity"
    封堵了"太简单不需要"路径
  ✗ "until design is approved" 封堵了"后补"路径

  → 唯一的出口：用户显式说"跳过设计"
  → 裁量空间趋近于零
  → 但设计上限 = 用户意志
```

## 梯度图

```
  裁量空间
  ────────
  100% ┤
       │  ████████████  Level 0: "可以考虑写测试"
   90% ┤
       │  ██████████
   80% ┤
       │  ████████
   70% ┤
       │  ██████      Level 1: "先写测试"
   60% ┤
       │  ████
   50% ┤
       │  ███         Level 2: "必须先写测试"
   40% ┤
       │  ██
   30% ┤
       │  █           Level 3: "绝不先写代码"
   20% ┤
       │  ▌           Level 4: "NO CODE WITHOUT TEST FIRST"
   10% ┤
       │  ▏           Level 5: <HARD-GATE> + EVERY
    1% ┤
       └──┬────┬────┬────┬────┬────┬──
         L0   L1   L2   L3   L4   L5

  每一级缩减约 50-70% 的裁量空间
  → 乘法衰减，不是线性衰减
  → 5 级后，裁量空间从 90% 降到 1%
  → 效果 = 0.9 × 0.67 × 0.5 × 0.33 × 0.3 × 0.03 ≈ 0.001
```
