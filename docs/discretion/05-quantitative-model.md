# 05 量化模型

裁量空间可以被近似量化。核心模型：DS = ∏(1 - s_i)，各路径封堵强度的乘积。

## 基础模型

```
  ┌──────────────────────────────────────────────────────────┐
  │  DS = ∏(1 - s_i)  对所有封堵路径 i                     │
  │                                                          │
  │  DS = Discretion Space（裁量空间）                        │
  │  s_i = 路径 i 的封堵强度（0 = 未封堵, 1 = 完全封堵）   │
  │                                                          │
  │  直觉：                                                  │
  │  - 未封堵路径 = (1 - 0) = 1.0 → 100% 通过              │
  │  - 完全封堵路径 = (1 - 1) = 0 → 乘积归零               │
  │  - 部分封堵路径 = (1 - 0.8) = 0.2 → 20% 通过           │
  │                                                          │
  │  模型假设：AI 绕过指令需要所有未封堵路径同时可通行。     │
  │  → 乘法效应：每增加一个封堵，裁量空间指数缩小           │
  └──────────────────────────────────────────────────────────┘
```

## 乘法效应 vs 加法效应

```
  ┌──────────────────────────────────────────────────────────┐
  │  加法模型（错误直觉）：                                  │
  │  封堵 3 条路径 → 减少 50% 裁量空间                      │
  │  封堵 6 条路径 → 减少 100% 裁量空间                     │
  │  → 暗示 6 条封堵后裁量空间为零                          │
  │  → 暗示前几条封堵效果最大，后面的越来越没用              │
  │                                                          │
  │  乘法模型（实际行为）：                                  │
  │  每条路径的封堵是独立的概率过滤器                        │
  │  → AI 需要通过所有未封堵路径才能绕过                    │
  │  → 通过概率 = 各路径通过概率的乘积                      │
  │  → 封堵越多，乘积越小 → 裁量空间指数缩小               │
  │                                                          │
  │  数值对比（假设每条路径封堵强度 0.8）：                  │
  │                                                          │
  │  封堵数  加法模型     乘法模型                           │
  │  ────    ────────     ────────                           │
  │  0       100%        100%                                │
  │  1       83%         20%                                 │
  │  2       67%         4%                                  │
  │  3       50%         0.8%                                │
  │  4       33%         0.16%                               │
  │  5       17%         0.032%                              │
  │  6       0%          0.0064%                             │
  │                                                          │
  │  → 乘法模型下，3 条封堵就把裁量空间压到 1% 以下         │
  │  → 6 条封堵把裁量空间压到 0.006%                        │
  │  → 这就是为什么 Superpowers 不只封堵一两条路径            │
  └──────────────────────────────────────────────────────────┘
```

## 不同强度级别的 DS 计算

```
  ┌──────────────────────────────────────────────────────────┐
  │  Level 0 (建议性)：                                      │
  │  无封堵 → s = [0, 0, 0, 0, 0, 0]                       │
  │  DS = 1.0 × 1.0 × ... = 1.0 → ~90%                     │
  │  （AI 几乎完全自由，但 90% 而非 100%                    │
  │   因为 AI 仍有基本服从的倾向）                           │
  │                                                          │
  │  Level 2 (命令性)：                                      │
  │  s = [0.7, 0, 0, 0, 0, 0]                               │
  │  DS = 0.3 × 1.0 × ... = 0.3 → ~30%                     │
  │  （降级路径被 "MUST" 封堵 70%）                         │
  │                                                          │
  │  Level 3 (否定性)：                                      │
  │  s = [0.95, 0.9, 0.5, 0, 0, 0]                          │
  │  DS = 0.05 × 0.1 × 0.5 = 0.0025 → ~10%                 │
  │  （降级+例外+部分时序被封堵）                            │
  │                                                          │
  │  Level 4 (铁律性)：                                      │
  │  s = [0.95, 0.9, 0.95, 0.8, 0, 0]                       │
  │  DS = 0.05 × 0.1 × 0.05 × 0.2 = 0.00005 → ~3%          │
  │  （降级+例外+后补+重定义被封堵）                         │
  │                                                          │
  │  Level 6 (道德性 + 全封堵)：                              │
  │  s = [0.95, 0.95, 0.95, 0.8, 0.8, 0.9]                  │
  │  DS = 0.05×0.05×0.05×0.2×0.2×0.1                        │
  │     = 0.0000005 → ~0.00005%                              │
  │  （六重封堵，裁量空间趋近于零）                          │
  └──────────────────────────────────────────────────────────┘
```

## 上下文衰减

```
  ┌──────────────────────────────────────────────────────────┐
  │  DS 不是常数——它随对话长度增长。                         │
  │                                                          │
  │  原因：指令的注意力权重随距离衰减                        │
  │  → 距离越远 → 注意力越低 → 封堵强度 s_i 下降            │
  │  → DS = ∏(1 - s_i(t)) 其中 s_i(t) 随时间 t 衰减        │
  │                                                          │
  │  衰减模型（假设指数衰减）：                               │
  │  s_i(t) = s_i(0) × e^(-λt)                              │
  │  λ = 衰减率（取决于模型和上下文长度）                    │
  │                                                          │
  │  短对话 (t 小)：s_i(t) ≈ s_i(0) → DS ≈ 设计值          │
  │  长对话 (t 大)：s_i(t) << s_i(0) → DS >> 设计值         │
  │                                                          │
  │  示例（假设 λ = 0.01/轮）：                               │
  │  第 1 轮:  s = 0.95 → DS ≈ 0.05                         │
  │  第 10 轮: s = 0.86 → DS ≈ 0.14                         │
  │  第 50 轮: s = 0.61 → DS ≈ 0.39                         │
  │  第 100 轮: s = 0.37 → DS ≈ 0.63                        │
  │                                                          │
  │  → 铁律在 100 轮后可能只剩下原始封堵强度的 37%          │
  │  → 裁量空间从 ~3% 回升到 ~63%                           │
  │  → 这是 Superpowers 最难解决的问题之一                   │
  │                                                          │
  │  Superpowers 的缓解（参见 gradient/06-anchoring.md）：    │
  │  - 技能重载 → 重置 s_i(t) 到 s_i(0)                     │
  │  - 清单追踪 → 在近处持续提醒                            │
  │  - 子智能体 → 隔离环境（t 永远从 0 开始）               │
  └──────────────────────────────────────────────────────────┘
```

## 模型的局限

```
  ┌──────────────────────────────────────────────────────────┐
  │  这个量化模型有几个重要局限：                            │
  │                                                          │
  │  1. s_i 不可测量                                        │
  │     → 封堵强度是推测值，不是实验数据                    │
  │     → 不同模型、不同语境下 s_i 可能差异很大             │
  │                                                          │
  │  2. 路径不独立                                          │
  │     → 模型假设路径独立，但实际路径可能相互影响           │
  │     → 封堵一条路径可能让 AI 更积极地寻找其他路径        │
  │     → 乘法模型可能高估或低估实际效果                    │
  │                                                          │
  │  3. 新路径可能涌现                                      │
  │     → 模型只考虑了已知的 6 条路径                       │
  │     → AI 可能发现设计者没想到的新路径                    │
  │     → 模型无法预测未知的虫洞                            │
  │                                                          │
  │  4. 衰减率 λ 不确定                                     │
  │     → 上下文衰减的速率取决于模型实现                    │
  │     → 不同模型（GPT-4, Claude, Gemini）衰减率不同       │
  │     → 同一模型不同版本衰减率可能变化                    │
  │                                                          │
  │  模型的价值：                                            │
  │  - 不是精确预测工具                                      │
  │  - 是概念框架——帮助理解"为什么多重封堵比单一封堵强"   │
  │  - 乘法效应是核心洞察——这是经验上可验证的              │
  │  - 具体数值不重要，结构关系（乘法 > 加法）才重要        │
  └──────────────────────────────────────────────────────────┘
```
