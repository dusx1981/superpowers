# 05 梯度策略

梯度不是越强越好。全程最强 = 信号噪声比下降 = 整体效果变差。梯度的策略性使用比单纯加强更重要。

## 5.1 金字塔分布：强少弱多

### Superpowers 中的强度分布

```
  ┌──────────────────────────────────────────────────────────┐
  │                                                          │
  │  Level 6 (道德性)   ┃  "Skip = lying"                    │
  │                     ┃  (验证技能，2 处)                   │
  │                     ┃                                    │
  │  Level 5 (结构性)   ┃  <EXTREMELY_IMPORTANT>             │
  │                     ┃  <HARD-GATE>                        │
  │                     ┃  (bootstrap + 门禁，3 处)           │
  │                     ┃                                    │
  │  Level 4 (铁律性)   ┃  NO CODE WITHOUT TEST              │
  │                     ┃  NO COMPLETION WITHOUT EVIDENCE     │
  │                     ┃  NO FIXES WITHOUT ROOT CAUSE        │
  │                     ┃  (3-4 条铁律)                       │
  │                     ┃                                    │
  │  Level 3 (否定性)   ┃  Never skip reviews                │
  │                     ┃  Never ignore escalation            │
  │                     ┃  Never trust agent reports          │
  │                     ┃  (20+ 处)                           │
  │                     ┃                                    │
  │  Level 2 (命令性)   ┃  You MUST create a task            │
  │                     ┃  You MUST invoke the skill          │
  │                     ┃  (10+ 处)                           │
  │                     ┃                                    │
  │  Level 1 (指示性)   ┃  "Check files, docs, commits"      │
  │                     ┃  "Prefer multiple choice"           │
  │                     ┃  (50+ 处)                           │
  │                     ┃                                    │
  │  Level 0 (建议性)   ┃  "Consider using..."               │
  │                     ┃  "User preferences override"        │
  │                     ┃  (5-10 处)                          │
  │                                                          │
  └──────────────────────────────────────────────────────────┘

  分布形态：金字塔形
  - 越强的指令越少
  - 越弱的指令越多
  - 这保证了高强度信号的稀缺性 → 不脱敏
```

### 为什么不是均匀分布？

```
  ┌──────────────────────────────────────────────────────────┐
  │  假设所有指令都是 Level 6 道德性否定：                   │
  │                                                          │
  │  <EXTREMELY-IMPORTANT>                                   │
  │  NEVER write code without tests FIRST                    │
  │  </EXTREMELY-IMPORTANT>                                  │
  │                                                          │
  │  <EXTREMELY-IMPORTANT>                                   │
  │  NEVER skip design review FIRST                          │
  │  </EXTREMELY-IMPORTANT>                                  │
  │                                                          │
  │  <EXTREMELY-IMPORTANT>                                   │
  │  NEVER commit without verification FIRST                 │
  │  </EXTREMELY-IMPORTANT>                                  │
  │                                                          │
  │  <EXTREMELY-IMPORTANT>                                   │
  │  NEVER skip clarifying questions FIRST                   │
  │  </EXTREMELY-IMPORTANT>                                  │
  │                                                          │
  │  ... 20 more of these ...                                │
  │                                                          │
  │  问题：                                                  │
  │  - 当一切都是 "EXTREMELY IMPORTANT" → 没有什么是特殊的  │
  │  - 信号噪声比下降——AI 无法区分真正关键 vs 一般重要     │
  │  - 类似于"狼来了"效应——每条都喊极重要 → AI 开始脱敏   │
  │  - 全程最强 = 全程噪声                                  │
  │                                                          │
  │  类比：                                                  │
  │  - 如果所有交通灯都是红灯 → 司机无法判断哪条路更危险   │
  │  - 如果所有邮件都标"紧急" → 没有邮件是紧急的          │
  │  - 如果所有指令都是铁律 → 没有铁律是特殊的              │
  └──────────────────────────────────────────────────────────┘
```

### 梯度差 = 优先级信号

```
  ┌──────────────────────────────────────────────────────────┐
  │  不同强度之间的差异本身就是信息：                        │
  │                                                          │
  │  Level 4: "NO CODE WITHOUT TEST FIRST"                   │
  │  Level 1: "Prefer multiple choice questions"             │
  │                                                          │
  │  AI 读到这两条时：                                       │
  │  - Level 4 的格式、语用、训练关联都告诉它"不可绕过"    │
  │  - Level 1 的格式、语用、训练关联都告诉它"建议性的"    │
  │  - 两者之间的差异 = "TDD 比多选题格式更重要"           │
  │                                                          │
  │  如果两条都是 Level 4：                                  │
  │  - AI 无法判断哪个更重要                                 │
  │  - 当两者冲突时（比如为了赶多选题格式而跳过 TDD），     │
  │    AI 不知道该优先哪个                                   │
  │  - 梯度差 = 优先级信息 = 冲突时的决策依据              │
  │                                                          │
  │  推论：                                                  │
  │  梯度不只是"让 AI 服从"的工具，                         │
  │  还是"告诉 AI 什么更重要"的信号系统。                  │
  │  梯度差 = 优先级信号。                                   │
  └──────────────────────────────────────────────────────────┘
```

## 5.2 反用降弱：何时故意使用弱指令

### 故意降弱的三个场景

```
  ┌──────────────────────────────────────────────────────────┐
  │  场景 1: 用户偏好不应被强制                              │
  │  ──────────────────────────────                          │
  │  "User preferences for spec location                     │
  │   override this default"                                │
  │                                                          │
  │  → 没有全大写，没有标签，没有否定                        │
  │  → 因为用户偏好应该有裁量空间                            │
  │  → 如果用 <HARD-GATE> 包裹 → 用户失去了选择权          │
  │  → 降弱 = 保护用户自主权                                 │
  │                                                          │
  │  场景 2: 防止无限循环                                    │
  │  ──────────────────                                      │
  │  "Fix any issues inline. No need to re-review            │
  │   — just fix and move on."                               │
  │                                                          │
  │  → "just fix and move on" 故意轻松                       │
  │  → 如果这里用 MANDATORY 或 NEVER SKIP → AI 可能         │
  │    在自审中无限循环：发现问题 → 修复 → 再自审 →         │
  │    发现新问题 → 修复 → ...                               │
  │  → 降弱 = 防止过度执行                                   │
  │                                                          │
  │  场景 3: 鼓励提问                                        │
  │  ────────────                                            │
  │  "It's always OK to pause and clarify.                   │
  │   Don't guess or make assumptions."                      │
  │                                                          │
  │  → "It's always OK" 是许可式，不是命令式                │
  │  → 如果用 MANDATORY ASK QUESTIONS → 子智能体可能        │
  │    强迫自己提问（即使不需要），拖慢速度                  │
  │  → 许可式降低了提问的心理门槛，                          │
  │    但不强制每事必问                                       │
  │  → 降弱 = 留裁量空间给 AI                                │
  └──────────────────────────────────────────────────────────┘
```

### 降弱的原则

```
  ┌──────────────────────────────────────────────────────────┐
  │  何时用强，何时用弱？决策原则：                          │
  │                                                          │
  │  用强（高梯度）：                                        │
  │  - AI 可能绕过的行为（跳过测试、忽略审阅）              │
  │  - 违反后果严重的行为（未验证就声称完成）               │
  │  - AI 有自然倾向反着做的行为（不写测试先写代码）        │
  │  - 流程中的关键门控点（设计审批、验证步骤）             │
  │                                                          │
  │  用弱（低梯度）：                                        │
  │  - AI 自然会做的行为（检查文件、问问题）                │
  │  - 用户应该有选择权的行为（偏好设置、执行方式）         │
  │  - 过度执行可能导致问题的行为（无限自审、强迫提问）     │
  │  - 具体实现细节（留裁量空间给 AI 的专业判断）           │
  │                                                          │
  │  核心判断：                                              │
  │  如果 AI 的"自然倾向"和"期望行为"一致 → 弱指令       │
  │  如果 AI 的"自然倾向"和"期望行为"相反 → 强指令       │
  │                                                          │
  │  梯度 = 纠偏力                                           │
  │  偏差越大，需要的纠偏力越强                              │
  │  没有偏差，不需要纠偏力                                  │
  └──────────────────────────────────────────────────────────┘
```

## 5.3 梯度的"预算"概念

```
  ┌──────────────────────────────────────────────────────────┐
  │  可以把梯度想象成"注意力预算"：                          │
  │                                                          │
  │  AI 在一个会话中能投入的"服从努力"是有限的。            │
  │  每条指令都在消耗这个预算。                              │
  │                                                          │
  │  预算分配：                                              │
  │                                                          │
  │  Level 6 指令：消耗预算 30%                              │
  │  → 但只有 2 条 → 总消耗 60%                             │
  │                                                          │
  │  Level 4 指令：消耗预算 15%                              │
  │  → 有 4 条 → 总消耗 60%                                 │
  │                                                          │
  │  Level 2 指令：消耗预算 5%                               │
  │  → 有 10 条 → 总消耗 50%                                │
  │                                                          │
  │  Level 1 指令：消耗预算 2%                               │
  │  → 有 50 条 → 总消耗 100%                               │
  │                                                          │
  │  如果所有指令都是 Level 6：                              │
  │  60 条 × 30% = 1800% → 预算严重超支                     │
  │  → AI 开始选择性忽略 → 哪些被忽略变得不可预测           │
  │                                                          │
  │  金字塔分布：                                             │
  │  2×30% + 4×15% + 10×5% + 50×2% = 60+60+50+100 = 270%  │
  │  → 仍然超支，但高强度指令的占比低                       │
  │  → AI 至少会对最关键的指令保持服从                      │
  │                                                          │
  │  注意：这是比喻，不是精确计算。                          │
  │  AI 没有真正的"预算"，但"注意力竞争"的效应是真实的。  │
  └──────────────────────────────────────────────────────────┘
```

## 5.4 Superpowers 中的梯度使用统计

```
  ┌──────────────────────────────────────────────────────────┐
  │  按 Level 统计 Superpowers 中的指令数量（估算）：       │
  │                                                          │
  │  Level 6:  2 处   (道德性否定)                          │
  │  Level 5:  3 处   (XML结构性约束)                       │
  │  Level 4:  4 处   (铁律)                                │
  │  Level 3: 25 处   (Never/Do NOT)                        │
  │  Level 2: 15 处   (You MUST / MANDATORY)                │
  │  Level 1: 60+ 处  (指示性)                              │
  │  Level 0:  8 处   (建议性)                              │
  │                                                          │
  │  总计约 117+ 条指令                                     │
  │                                                          │
  │  Level 4+ 占比：9/117 ≈ 8%                              │
  │  Level 3+ 占比：34/117 ≈ 29%                            │
  │  Level 2+ 占比：49/117 ≈ 42%                            │
  │                                                          │
  │  → 不到 10% 的指令是"铁律级"                           │
  │  → 约 30% 的指令是"否定级"                             │
  │  → 约 60% 的指令是"建议/指示级"                        │
  │                                                          │
  │  这符合金字塔分布——强少弱多                             │
  │  高强度指令的稀缺性保证了它们不被忽视                   │
  └──────────────────────────────────────────────────────────┘
```
