# 02 格式层

格式层是梯度三层的中间层——通过文本的视觉和结构格式差异，在 AI 的注意力机制中获得不同权重。格式不改变语义，但改变 AI 对语义的处理优先级。

## 2.1 对比度驱动注意力

```
  ┌──────────────────────────────────────────────────────────┐
  │  想象 AI 的注意力是一个信号处理系统：                    │
  │                                                          │
  │  输入：一长串小写文本 + 少量大写/标签                    │
  │                                                          │
  │  ┌──────────────────────────────────────────────┐       │
  │  │ make sure you write tests first. always      │       │
  │  │ follow the code review process. check that   │       │
  │  │ all tests pass before committing. NO CODE    │       │
  │  │ WITHOUT TEST FIRST                           │       │
  │  └──────────────────────────────────────────────┘       │
  │       │                                                  │
  │       ▼                                                  │
  │  ┌──────────────────────────────────────────────┐       │
  │  │  注意力权重分布（示意）：                      │       │
  │  │                                                │       │
  │  │  make sure you write tests first ████          │       │
  │  │  always follow code review     ██████          │       │
  │  │  check that all tests pass     ████            │       │
  │  │  NO CODE WITHOUT TEST FIRST    ███████████████ │       │
  │  │                                ▲               │       │
  │  │                                │               │       │
  │  │                          格式对比度驱动        │       │
  │  │                          注意力加压            │       │
  │  └──────────────────────────────────────────────┘       │
  │                                                          │
  │  全大写文本在视觉和 token 层面都与周围形成对比          │
  │  对比度 → 注意力权重增加 → 更难被忽略                  │
  └──────────────────────────────────────────────────────────┘
```

### 对比度的三个来源

```
  ┌──────────────────────────────────────────────────────────┐
  │  1. 大小写对比                                           │
  │     小写环境中出现全大写 → 视觉跳变                     │
  │     "always follow" vs "NO CODE WITHOUT"                 │
  │     → 类似于文章中突然加粗——视觉优先处理               │
  │                                                          │
  │  2. 结构对比                                             │
  │     纯文本中出现代码块/标签 → 结构跳变                 │
  │     普通段落 vs ```代码块``` vs <XML标签>               │
  │     → 结构边界处注意力自然集中                          │
  │                                                          │
  │  3. 密度对比                                             │
  │     流畅段落中出现短句/独立行 → 密度跳变               │
  │     5 行解释 vs "NO PRODUCTION CODE WITHOUT              │
  │     A FAILING TEST FIRST"                                │
  │     → 短而强的文本在长文本中更突出                      │
  └──────────────────────────────────────────────────────────┘
```

## 2.2 XML 标签的结构性注意力

XML 标签比 ALL CAPS 更强，因为它不仅改变格式，还引入了元信息层。

### XML 标签的三重加权

```
  ┌──────────────────────────────────────────────────────────┐
  │  1. 边界标记                                             │
  │  ────────                                                │
  │  <HARD-GATE> ... </HARD-GATE>                            │
  │  → 有明确的开始和结束                                    │
  │  → AI 知道"这段内容是特殊区域"                          │
  │  → 边界内外的内容获得不同的注意力权重                   │
  │                                                          │
  │  2. 语义标签名                                           │
  │  ────────────                                            │
  │  标签名本身携带语义：                                    │
  │  <note>         → "这是注释"                            │
  │  <important>    → "这很重要"                            │
  │  <HARD-GATE>    → "这是硬门禁"                          │
  │  <EXTREMELY_IMPORTANT> → "这极端重要"                   │
  │                                                          │
  │  → 标签名在 AI 解析时被当作元信息                       │
  │  → 元信息比正文权重更高                                 │
  │  → 标签名的语义本身就在施加权重                         │
  │                                                          │
  │  3. 嵌套叠加                                            │
  │  ──────────                                              │
  │  <EXTREMELY_IMPORTANT>                                   │
  │    <EXTREMELY-IMPORTANT>                                 │
  │      [内容]                                              │
  │    </EXTREMELY-IMPORTANT>                                │
  │  </EXTREMELY_IMPORTANT>                                  │
  │                                                          │
  │  → 外层标签 + 内层标签 = 双重元信息                     │
  │  → AI 处理时，两层元信息都指向"不可忽略"               │
  │  → 嵌套不是冗余，是双重加压                             │
  └──────────────────────────────────────────────────────────┘
```

### 标签名的语义强度梯度

```
  标签名的选择本身就是格式层 + 语用层的协同：

  <note>                    → 低——"注意"（建议级）
  <important>               → 中——"重要"（关注级）
  <important-reminder>      → 中高——"重要提醒"（行动级）
  <HARD-GATE>               → 高——"硬门禁"（禁止级）
  <EXTREMELY_IMPORTANT>     → 最高——"极端重要"（不可绕过级）
  <SUBAGENT-STOP>           → 特殊——"子智能体停下"（角色级）

  标签名越长、越极端，权重越高：
  - <note>：4 个字符 → 低权重标签
  - <EXTREMELY_IMPORTANT>：21 个字符 → 高权重标签
  - 字符长度本身也是格式信号——更长的标签名 = 更不寻常 = 更多注意力
```

### 嵌套的实际结构

```
  OpenCode 注入的实际结构：

  <EXTREMELY_IMPORTANT>              ← 外层（第 1 重：标签边界）
    You have superpowers.
    "ALREADY LOADED" 声明             ← 第 2 重：状态声明

    <EXTREMELY-IMPORTANT>            ← 内层（第 3 重：嵌套标签）
      ABSOLUTELY MUST invoke          ← 第 4 重：绝对命令
      DO NOT HAVE A CHOICE           ← 第 5 重：取消裁量
      Not negotiable. Not optional.  ← 第 6 重：连续否定
    </EXTREMELY-IMPORTANT>

    [技能内容]
    [工具映射]
  </EXTREMELY_IMPORTANT>

  六重加压——AI 要绕过，必须同时忽略六个层级的权重信号

  注意：外层用下划线 EXTREMELY_IMPORTANT，
  内层用连字符 EXTREMELY-IMPORTANT。
  微小的格式差异 → 不是笔误，而是让两个标签在 token 层面
  不完全相同 → 避免被 AI 当作"同一个标签重复"而降权。
```

## 2.3 代码块格式

### 为什么铁律用代码块包裹？

```
  不用代码块：
  NO PRODUCTION CODE WITHOUT A FAILING TEST FIRST

  用代码块：
  ```
  NO PRODUCTION CODE WITHOUT A FAILING TEST FIRST
  ```

  代码块的三个效果：
  ────────────────
  1. 结构隔离
     代码块与周围文本在渲染上有明确边界
     → AI 把它当作"独立单元"处理，而非"段落的一部分"

  2. 等宽字体心理暗示
     在 AI 的训练语料中，代码块通常包含：
     - 代码（精确、不可随意修改）
     - 命令行输出（事实、不可争辩）
     - 配置文件（规则、必须遵守）
     → 代码块 = 精确/事实/规则——不可重新解读

  3. 不容改写
     代码块中的内容在 AI 的心理模型中更倾向于"引用"而非"描述"
     → 引用不可改写，描述可以改写
     → 铁律被当作引用 → 不可改写
```

## 2.4 表格的结构化对比

### 表格 vs 列表的权重差异

```
  列表格式：
  ────────
  - "太简单不需要测试"：简单代码也会坏
  - "我之后再测"：之后通过的测试什么也证明不了

  → 线性阅读，AI 可能跳过某些条目
  → 借口和反驳混在一起，视觉权重相同

  表格格式：
  ────────
  | Thought                      | Reality                           |
  |------------------------------|-----------------------------------|
  | "Too simple to test"        | Simple code breaks. 30 seconds.   |
  | "I'll test after"            | Tests passing immediately         |
  |                              | prove nothing.                    |

  → 左右对比，视觉冲击强
  → 借口在左、反驳在右——因果关系视觉化
  → 表格结构本身暗示"这些借口都有反驳"
  → AI 读到左列时，右列的否决已经等待
```

### Good/Bad 标签的视觉评价

```
  <Good>
  ```typescript
  test('retries failed operations 3 times', async () => { ... });
  ```
  Clear name, tests real behavior, one thing
  </Good>

  <Bad>
  ```typescript
  test('retry works', async () => { ... });
  ```
  Vague name, tests mock not code
  </Bad>

  评价嵌入在结构中：
  - 不是"推荐写法"和"不推荐写法"这样的描述性标题
  - 而是直接用 Good/Bad 标注
  - 标签名即价值判断 → AI 不需要"理解为什么不推荐"
  - 标签名直接告诉 AI "这是好的/坏的"
```

## 2.5 格式层的脆弱性

```
  ┌──────────────────────────────────────────────────────────┐
  │  格式层是三层中最脆弱的：                                │
  │                                                          │
  │  1. Tokenizer 依赖                                       │
  │     ALL CAPS 有效的前提是 tokenizer 区分大小写           │
  │     如果未来模型使用不区分大小写的 tokenizer，           │
  │     "MUST" 和 "must" 产生相同的 token → 格式层失效      │
  │                                                          │
  │  2. 渲染环境依赖                                         │
  │     表格、代码块的效果依赖渲染环境                       │
  │     纯文本终端 vs Markdown 渲染器 → 视觉差异巨大        │
  │     AI 的"视觉"是 token 序列，不是渲染结果              │
  │     → 某些格式差异在 token 层面可能被拉平               │
  │                                                          │
  │  3. 习惯化风险                                           │
  │     如果 AI 在同一会话中反复看到 <EXTREMELY_IMPORTANT>   │
  │     → 格式的"异常性"降低                                │
  │     → 对比度下降                                         │
  │     → 注意力加权减弱                                     │
  │                                                          │
  │  防御：                                                  │
  │  - 格式层不单独依赖，与语用层和训练层协同                │
  │  - 高强度格式（Level 5-6）只在关键处使用（稀缺性）      │
  │  - 不同场景使用不同格式标签（多样性防习惯化）           │
  └──────────────────────────────────────────────────────────┘
```
