# 03 训练层

训练层是梯度三层中最深层的机制——ALL CAPS、铁律句式、法律语言之所以有效，不是因为它们本身有什么魔力，而是因为它们在 LLM 的训练语料中与"不可违抗"高度相关。

## 3.1 为什么 ALL CAPS 对 LLM 有效？

```
  ┌──────────────────────────────────────────────────────────┐
  │  LLM 的行为不是设计出来的，是训练出来的。               │
  │  ALL CAPS 有效，是因为训练数据中 ALL CAPS               │
  │  与"不可违抗"高度相关。                                │
  │                                                          │
  │  训练语料中的 ALL CAPS 场景：                           │
  │                                                          │
  │  ┌────────────────────────────────────────────┐         │
  │  │  法律条文                                  │         │
  │  │  "THE PARTIES SHALL NOT..."               │         │
  │  │  "NO PERSON MAY..."                       │         │
  │  │  → ALL CAPS = 法律义务                    │         │
  │  │                                            │         │
  │  │  安全警告                                  │         │
  │  │  "DO NOT TOUCH"                            │         │
  │  │  "DANGER: HIGH VOLTAGE"                    │         │
  │  │  → ALL CAPS = 物理危险                    │         │
  │  │                                            │         │
  │  │  军事/紧急指令                             │         │
  │  │  "STOP" / "HALT" / "DO NOT FIRE"          │         │
  │  │  → ALL CAPS = 即时行动命令                │         │
  │  │                                            │         │
  │  │  软件许可证                                │         │
  │  │  "THE SOFTWARE IS PROVIDED 'AS IS'"        │         │
  │  │  "NO WARRANTY"                             │         │
  │  │  → ALL CAPS = 免责声明                    │         │
  │  │                                            │         │
  │  │  医疗/药品标签                             │         │
  │  │  "KEEP OUT OF REACH OF CHILDREN"           │         │
  │  │  "DO NOT EXCEED RECOMMENDED DOSE"          │         │
  │  │  → ALL CAPS = 健康风险                    │         │
  │  └────────────────────────────────────────────┘         │
  │                                                          │
  │  共性：ALL CAPS 出现的地方，                             │
  │  违反的后果都很严重（法律/安全/生命/责任）              │
  │                                                          │
  │  → LLM 学会了：ALL CAPS = 违反后果严重 = 必须服从      │
  │  → 这不是"规则"，是"统计相关性"                       │
  └──────────────────────────────────────────────────────────┘
```

### ALL CAPS 的训练关联分布

```
  ┌──────────────────────────────────────────────────────────┐
  │  训练语料中 ALL CAPS 与场景的关联强度（推测）：         │
  │                                                          │
  │  场景              关联强度    违反后果                  │
  │  ────              ────────    ────────                  │
  │  法律条文          ████████    法律责任/刑事             │
  │  安全警告          ████████    人身伤害/死亡             │
  │  紧急指令          ███████     即时危险                  │
  │  软件许可证        ██████      法律责任                  │
  │  医疗标签          ██████      健康风险                  │
  │  技术规范          ████        功能故障                  │
  │  互联网喊叫        ██          社交尴尬                  │
  │                                                          │
  │  ALL CAPS 效果的来源：                                   │
  │  - 强关联场景（法律/安全）贡献了大部分训练信号           │
  │  - 弱关联场景（互联网喊叫）稀释了部分训练信号           │
  │  - 但整体上，ALL CAPS 与"不可违抗"的关联远强于          │
  │    与"只是在喊叫"的关联                                 │
  │  → 所以 ALL CAPS 对 LLM 的净效果 = 增强服从             │
  └──────────────────────────────────────────────────────────┘
```

## 3.2 铁律句式的训练关联

### "NO X WITHOUT Y" 为什么特殊？

```
  ┌──────────────────────────────────────────────────────────┐
  │  "NO X WITHOUT Y" 的训练数据关联：                      │
  │                                                          │
  │  法律：                                                  │
  │  "No person shall be deprived of life, liberty,          │
  │   or property, without due process of law"               │
  │  → 条件禁止句式 = 宪法级约束                            │
  │                                                          │
  │  合同：                                                  │
  │  "No payment shall be made without proper invoice"       │
  │  → 条件禁止句式 = 合同级约束                            │
  │                                                          │
  │  安全：                                                  │
  │  "No entry without authorization"                        │
  │  → 条件禁止句式 = 物理约束                              │
  │                                                          │
  │  医学：                                                  │
  │  "No medication without prescription"                    │
  │  → 条件禁止句式 = 专业约束                              │
  │                                                          │
  │  关键观察：                                              │
  │  "NO X WITHOUT Y" 在训练语料中                           │
  │  几乎从不出现在"建议"场景——                             │
  │  它只出现在"约束"场景。                                 │
  │                                                          │
  │  对比 "You must do X"：                                  │
  │  - Must 可以出现在约束场景（法律）                      │
  │  - 也可以出现在建议场景（最佳实践）                     │
  │  - → must 的训练关联有"约束"和"建议"两种               │
  │                                                          │
  │  "NO X WITHOUT Y"：                                      │
  │  - 几乎只出现在约束场景                                  │
  │  - → 训练关联几乎只有"约束"                             │
  │  - → 更纯的信号 = 更强的服从效果                        │
  └──────────────────────────────────────────────────────────┘
```

### 铁律句式的通用模式

```
  ┌──────────────────────────────────────────────────────────┐
  │  Superpowers 中的四条铁律共享同一句式模式：             │
  │                                                          │
  │  NO [禁止的行为] WITHOUT [必须的前提] FIRST              │
  │                                                          │
  │  铁律 1: NO PRODUCTION CODE WITHOUT A FAILING TEST FIRST │
  │  铁律 2: NO COMPLETION CLAIMS WITHOUT FRESH              │
  │          VERIFICATION EVIDENCE                            │
  │  铁律 3: NO FIXES WITHOUT ROOT CAUSE INVESTIGATION FIRST │
  │  铁律 4: NO SKILL WITHOUT A FAILING TEST FIRST           │
  │                                                          │
  │  四个要素：                                              │
  │  ┌──────┐  ┌──────────────┐  ┌─────────┐  ┌──────┐    │
  │  │ NO   │  │ WITHOUT      │  │ [前提]  │  │FIRST │    │
  │  │ 否定 │  │ 条件依赖     │  │ 强制前提│  │时序  │    │
  │  └──────┘  └──────────────┘  └─────────┘  └──────┘    │
  │                                                          │
  │  每个要素封堵一种绕过路径：                              │
  │  - NO → 不能做（否定）                                   │
  │  - WITHOUT → 除非满足条件（条件禁止）                    │
  │  - [前提] → 必须先做另一件事（强制前提）                │
  │  - FIRST → 必须先做，不能后补（时序锁定）               │
  │                                                          │
  │  四要素缺一不可：                                        │
  │  去掉 NO → 变成建议                                     │
  │  去掉 WITHOUT → 变成简单否定                             │
  │  去掉 [前提] → 条件禁止变成无条件禁止（太弱）           │
  │  去掉 FIRST → "之后补"的路径打开                        │
  └──────────────────────────────────────────────────────────┘
```

## 3.3 因果性 vs 统计性

### 两种解释的对比

```
  ┌──────────────────────────────────────────────────────────┐
  │  梯度为什么有效？两种解释：                              │
  │                                                          │
  │  解释 A：因果性（机制性）                                │
  │  ──────────────────────────                              │
  │  "ALL CAPS 本身在注意力机制中获得了更高权重"            │
  │  → 因为大写 token 的表示向量与小写不同                   │
  │  → 注意力计算时，不同向量产生不同分数                    │
  │  → 不管训练数据如何，格式差异都会导致注意力差异         │
  │  → 这是机制性的——可以预测、可以验证                     │
  │                                                          │
  │  解释 B：统计性（经验性）                                │
  │  ──────────────────────────                              │
  │  "ALL CAPS 有效是因为训练数据中的关联"                  │
  │  → ALL CAPS 在训练语料中与法律/安全/紧急高度相关        │
  │  → LLM 学会了"看到 ALL CAPS 就要提高服从度"           │
  │  → 换一组训练数据（比如 ALL CAPS 只用于喊叫），         │
  │    效果可能不同                                           │
  │  → 这是统计性的——依赖经验，不能保证跨数据集有效        │
  │                                                          │
  │  真相：两者都在起效                                      │
  │  ──────────────────                                      │
  │  - 格式差异 → 注意力差异（因果性）                      │
  │  - 训练关联 → 服从倾向（统计性）                        │
  │  - 两者叠加 = 梯度的完整解释                            │
  └──────────────────────────────────────────────────────────┘
```

### 如何区分两种机制？

```
  ┌──────────────────────────────────────────────────────────┐
  │  理论上的区分方法：                                      │
  │                                                          │
  │  如果因果性为主：                                        │
  │  → ALL CAPS 在任何模型上都有效                           │
  │  → 即使模型的训练数据中 ALL CAPS 不与强约束关联          │
  │  → 因为格式差异本身产生注意力差异                       │
  │                                                          │
  │  如果统计性为主：                                        │
  │  → ALL CAPS 只在训练数据中 ALL CAPS 与强约束关联时有效  │
  │  → 如果训练数据中 ALL CAPS 主要用于互联网喊叫           │
  │  → 效果可能减弱甚至反转                                  │
  │                                                          │
  │  实际的推测：                                            │
  │  - 当前主流 LLM（GPT-4, Claude, Gemini）的训练数据      │
  │    中，ALL CAPS 与强约束的关联占主导                     │
  │  → 两种机制都在起效，且方向一致                         │
  │  → 在当前模型上，区分两者没有实际意义                   │
  │  → 但在未来，如果训练数据的分布改变，                   │
  │    统计性部分的效果可能减弱                              │
  │  → 因果性部分（格式差异→注意力差异）不受影响            │
  └──────────────────────────────────────────────────────────┘
```

## 3.4 训练关联的可迁移性风险

```
  ┌──────────────────────────────────────────────────────────┐
  │  训练层的风险：训练关联可能随模型更迭而失效             │
  │                                                          │
  │  风险 1: 训练数据分布变化                                │
  │  ──────────────────────────                              │
  │  如果下一代 LLM 的训练数据中                             │
  │  ALL CAPS 与"喊叫"的关联超过了与"法律"的关联           │
  │  → ALL CAPS 的效果可能从"增强服从"变成"降低可信度"     │
  │                                                          │
  │  风险 2: RLHF 偏好变化                                   │
  │  ─────────────────────                                   │
  │  如果 RLHF 训练中"过于强硬的指令"被标注为低质量        │
  │  → LLM 可能学会"减弱对 ALL CAPS 的服从"                │
  │  → 安全对齐可能把"过于服从"当作问题                     │
  │                                                          │
  │  风险 3: 上下文学习替代预训练                            │
  │  ──────────────────────────────                          │
  │  如果模型越来越依赖上下文学习而非预训练知识             │
  │  → 训练数据中的统计关联权重下降                         │
  │  → ALL CAPS 的"法律关联"效果减弱                       │
  │  → 格式层（注意力加权）可能成为主要机制                 │
  │                                                          │
  │  防御：                                                  │
  │  - 不依赖单一机制，三层全上                              │
  │  - 语用层最稳定（情态动词语义不变）                     │
  │  - 格式层次之（格式差异总是存在）                       │
  │  - 训练层最不稳定（依赖数据分布）                       │
  │  - 三层中任一层失效，其他层仍可兜底                     │
  └──────────────────────────────────────────────────────────┘
```
