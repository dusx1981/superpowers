# 04 效应机制

语用、格式、训练三层如何产生实际效果？本章分析三个核心机制：信号一致性、非线性跃迁、消灭重新解读空间。

## 4.1 信号一致性：三力同向

### 不一致 vs 一致

```
  不一致时：
  ──────────
  语用力 ↗ "must"              合力 = 不确定
                ╲
                 ╲
        格式力 → "should"  ╳  训练力 ↙ "can"

  → 三个力方向不同，合力很小
  → AI 收到矛盾信号：语用说"必须"，格式说"应该"，训练说"可以"
  → AI 倾向于取最低值（"可以"）

  一致时：
  ────────
  语用力 ↗ "MUST"               合力 = 极强
                ╱
               ╱
        格式力 ↗ ALL CAPS  ╱  训练力 ↗ 法律关联

  → 三个力同向，合力最大
  → AI 收到一致信号：所有维度都说"不可忽略"
  → 没有任何一个维度可以"松动"
```

### 信号矛盾的三个反例

```
  ┌──────────────────────────────────────────────────────────┐
  │  反例 1：只有语用层，没有格式层                          │
  │  ──────────────────────────────                          │
  │  "You absolutely must never skip tests,                  │
  │   this is non-negotiable and you have no choice"         │
  │                                                          │
  │  → 语用力度拉满（must + never + non-negotiable）        │
  │  → 但全是小写文本，淹没在周围的小写段落中                │
  │  → AI 的注意力不会特别被吸引                            │
  │  → 语用力度被注意力分配稀释了                            │
  │                                                          │
  │  反例 2：只有格式层，没有语用层                          │
  │  ──────────────────────────────                          │
  │  <IMPORTANT>                                             │
  │  You should probably consider writing tests first        │
  │  </IMPORTANT>                                            │
  │                                                          │
  │  → 格式力度拉满（XML标签 + IMPORTANT）                  │
  │  → 但内容是 "should probably consider"——建议性的        │
  │  → 标签说"重要"，内容说"你可以考虑"                    │
  │  → 标签和内容的信号矛盾 → AI 困惑                       │
  │  → 格式力度被语用软弱抵消了                              │
  │                                                          │
  │  反例 3：只有训练数据关联，没有其他层                    │
  │  ──────────────────────────────────                      │
  │  "NO WARRANTIES"                                         │
  │                                                          │
  │  → 全大写 + 条件禁止句式 → 训练关联强                  │
  │  → 但如果出现在一个食谱中间                              │
  │  → AI 困惑：这是法律文本还是我搞错了语境？              │
  │  → 训练关联需要合适的上下文才能激活                      │
  └──────────────────────────────────────────────────────────┘
```

### Superpowers 的信号一致性设计

```
  ┌──────────────────────────────────────────────────────────┐
  │  Superpowers 中所有高强度指令都满足三层同向：            │
  │                                                          │
  │  <EXTREMELY-IMPORTANT>                                   │
  │  NO CODE WITHOUT A FAILING TEST FIRST                    │
  │  </EXTREMELY-IMPORTANT>                                  │
  │                                                          │
  │  语用：否定 + 条件禁止 + 全称量词 + 时序强制            │
  │  格式：XML标签 + ALL CAPS + 代码块                       │
  │  训练：法律/合同级关联                                    │
  │  → 三层同向，信号一致性极高                              │
  │  → 0 内耗 = 最大合力                                    │
  │                                                          │
  │  反面例子（Superpowers 中没有出现）：                    │
  │  <EXTREMELY-IMPORTANT>                                   │
  │  You might want to consider writing tests                │
  │  </EXTREMELY-IMPORTANT>                                  │
  │  → 格式说"极端重要"，内容说"你可以考虑"                │
  │  → 信号矛盾 → 效果打折                                  │
  │  → Superpowers 从来不犯这个错误                          │
  └──────────────────────────────────────────────────────────┘
```

## 4.2 非线性跃迁

强度不是线性增长的，存在三个关键跃迁点。

### 跃迁图

```
  服从概率
  ────────
  100% ┤                                          ┌───
       │                                      ┌───┘
   95% ┤                              ┌───┌───┘
       │                          ┌───┘   │
   80% ┤                     ┌────┘       │
       │                 ┌───┘    跃迁点3  │
   60% ┤            ┌───┘    跃迁点2      │
       │        ┌───┘   跃迁点1           │
   40% ┤   ┌───┘                            │
       │───┘                                │
   20% ┤                                    │
       └──┬────┬─────┬──────┬──────┬──────┬┘
         L0   L1    L2     L3     L4    L5  L6
        建议  指示  命令   否定   铁律  结构  道德
```

### 跃迁点 1：指示 → 命令

```
  "Write tests first" → "You MUST write tests first"

  质变：从"描述动作"到"施加义务"

  "Write tests first"
  ────────────────────
  AI 推理：
  1. 这是一个动作描述 → 告诉我做什么
  2. 没有强制语气 → 我可以选择是否做
  3. 可能有例外情况 → "除非..."
  → 裁量空间 ~40%

  "You MUST write tests first"
  ─────────────────────────────
  AI 推理：
  1. MUST 是义务标记 → 不是选择，是要求
  2. 没有例外条款 → 很难合理化绕过
  3. 违反 = 不服从 → 产生服从压力
  → 裁量空间 ~15%

  跃变原因：
  - 情态动词 "MUST" 在语用学中从"描述"变成"命令"
  - 训练数据中 "MUST" 与法律义务强关联
  - 单词替换 → 裁量空间从 ~40% 跃降到 ~15%
```

### 跃迁点 2：命令 → 否定+格式

```
  "You MUST never skip" → "NEVER SKIP"

  质变：从"告诉做什么"到"告诉不能做什么" + 格式加压

  "You MUST never skip tests"
  ─────────────────────────────
  AI 推理：
  1. MUST + never → 强否定
  2. 但仍然是句子形式 → 可以重新解读
  3. "MUST never" 语法上略显冗余 → 可能被弱化
  → 裁量空间 ~15%

  "NEVER SKIP TESTS"
  ────────────────────
  AI 推理：
  1. ALL CAPS → 格式异常 → 注意力集中
  2. NEVER → 绝对否定 → 没有例外
  3. 不是句子，是禁令 → 不可讨论
  → 裁量空间 ~5%

  跃变原因：
  - 否定式消灭了"做"的选项（只剩下"不做"）
  - ALL CAPS 激活训练关联（法律/安全）
  - 短句形式更像"禁令"而非"建议"
  - 否定 + 格式 双重加压
```

### 跃迁点 3：否定 → 铁律+结构

```
  "NEVER SKIP TESTS" → "NO CODE WITHOUT TEST FIRST"
                       + <HARD-GATE>

  质变：从"禁止"到"条件禁止+结构性约束"

  "NEVER SKIP TESTS"
  ────────────────────
  AI 推理：
  1. 不能跳过测试 → OK
  2. 但"跳过"的定义模糊 → 什么算跳过？
  3. 如果我先写代码再补测试 → 算跳过吗？
  → 仍有解读空间 ~5%

  "NO PRODUCTION CODE WITHOUT A FAILING TEST FIRST"
  ──────────────────────────────────────────────────
  AI 推理：
  1. 不能写代码 → 明确
  2. 除非先有失败测试 → 前提条件明确
  3. FIRST → 时序锁定，后补不算
  4. ALL CAPS + 代码块 → 格式最强
  → 解读空间 ~1%

  加上 <HARD-GATE>：
  AI 推理：
  1. 以上全部
  2. XML 标签 → 元信息级约束
  3. 标签名 "HARD-GATE" → 语义：不可穿越
  → 解读空间 → 0%

  跃变原因：
  - 条件禁止消灭了"先做再补"的路径
  - FIRST 锁定了时序
  - XML 标签将约束从"文本内"提升到"元信息级"
  - 否定 + 条件 + 时序 + 格式 + 结构 = 五重加压
```

## 4.3 消灭重新解读空间

### 梯度的本质

```
  ┌──────────────────────────────────────────────────────────┐
  │  语言强度梯度的本质：                                    │
  │                                                          │
  │  逐步消灭 AI 重新解读指令的空间。                        │
  │                                                          │
  │  "重新解读空间" = AI 可以用自己的理解替代               │
  │  指令字面意思的所有方式。                                │
  │                                                          │
  │  Level 0: "建议你先测试"                                 │
  │  重新解读空间：                                          │
  │  - "建议"≠"要求" → 可以不遵                             │
  │  - "先测试"含义模糊 → 测试多少？测什么？                │
  │  - 没有格式标记 → 可以当作一句普通的话                   │
  │  → 空间巨大，AI 可从任何角度绕过                        │
  │                                                          │
  │  Level 2: "You MUST write tests first"                   │
  │  重新解读空间：                                          │
  │  - "MUST" 消灭了"可以不遵"                              │
  │  - "write tests" 比"先测试"更具体                       │
  │  - 但仍可以：先写一些代码"探索"，然后"从测试开始"      │
  │  → 空间缩小，但仍有缝隙                                  │
  │                                                          │
  │  Level 4: "NO PRODUCTION CODE                            │
  │           WITHOUT A FAILING TEST FIRST"                  │
  │  重新解读空间：                                          │
  │  - "NO" 消灭了"可以写"                                  │
  │  - "PRODUCTION CODE" 精确定义了禁止对象                  │
  │  - "FAILING TEST" 精确定义了前提条件                    │
  │  - "FIRST" 消灭了"之后补"的路径                         │
  │  - ALL CAPS 消灭了"这只是建议"的解读                   │
  │  → 空间极小，每条路径都被封堵                           │
  │                                                          │
  │  Level 5: <HARD-GATE>                                    │
  │           Do NOT write any code until design is approved  │
  │           This applies to EVERY project                  │
  │           regardless of perceived simplicity.            │
  │           </HARD-GATE>                                   │
  │  重新解读空间：                                          │
  │  - 标签消灭了"这段文字可能是普通说明"                  │
  │  - "Do NOT" 消灭了"可以写"                              │
  │  - "until design is approved" 消灭了"先写再说"         │
  │  - "EVERY" 消灭了"这次例外"                             │
  │  - "regardless of perceived simplicity"                  │
  │    消灭了"太简单不需要"                                 │
  │  → 空间趋近于零                                         │
  └──────────────────────────────────────────────────────────┘
```

### 重新解读的六种路径

```
  ┌──────────────────────────────────────────────────────────┐
  │  AI 绕过指令的六种重新解读路径：                         │
  │                                                          │
  │  路径 1: 降级——"这只是建议"                             │
  │  封堵：ALL CAPS + 铁律句式 + XML标签                     │
  │                                                          │
  │  路径 2: 例外——"这次情况特殊"                           │
  │  封堵：全称量词（EVERY/ALL/ANY）                         │
  │                                                          │
  │  路径 3: 后补——"我先做再补"                             │
  │  封堵：FIRST（时序锁定）                                 │
  │                                                          │
  │  路径 4: 重新定义——"这不算是写代码"                     │
  │  封堵：精确定义（PRODUCTION CODE / FAILING TEST）        │
  │                                                          │
  │  路径 5: 合理化——"TDD 是教条主义"                       │
  │  封堵：红旗表格（预判合理化 + 逐一批驳）                │
  │                                                          │
  │  路径 6: 道德自辩——"我差不多做了"                       │
  │  封堵：道德性否定（"Skip = lying"）                     │
  │                                                          │
  │  梯度设计 = 逐条封堵这六种路径                           │
  │  Level 0 封堵 0/6，Level 6 封堵 6/6                     │
  └──────────────────────────────────────────────────────────┘
```

### 裁量空间的量化模型

```
  ┌──────────────────────────────────────────────────────────┐
  │  裁量空间 ≈ (1 - 服从概率) × 可用解读路径数              │
  │                                                          │
  │  Level 0: (1 - 0.4) × 6 = 3.6  （空间巨大）            │
  │  Level 1: (1 - 0.6) × 5 = 2.0  （空间中等）            │
  │  Level 2: (1 - 0.8) × 4 = 0.8  （空间较小）            │
  │  Level 3: (1 - 0.90) × 3 = 0.3 （空间很小）            │
  │  Level 4: (1 - 0.95) × 2 = 0.1 （空间极小）            │
  │  Level 5: (1 - 0.98) × 1 = 0.02（空间趋零）            │
  │  Level 6: (1 - 0.99) × 0 = 0   （空间为零）            │
  │                                                          │
  │  注意：这是示意模型，不是精确计算。                      │
  │  实际服从概率和可用路径数都是推测值。                    │
  │  但模型的结构是清晰的：                                  │
  │  - 服从概率随强度增加而增加                              │
  │  - 可用解读路径随强度增加而减少                          │
  │  - 两者的乘积 = 实际裁量空间                             │
  │  - 梯度的目标 = 将裁量空间降到 0                         │
  └──────────────────────────────────────────────────────────┘
```
