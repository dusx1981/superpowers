# Superpowers 智能体架构分析

Superpowers 的智能体体系不是一组独立的 Agent 类，而是**由技能驱动的角色化子智能体系统**。每个智能体都是一个带有精确指令的 Task dispatch，由 Controller（主会话）在特定技能的指导下动态创建和协调。

## 关键洞察

1. **没有类继承，只有 Prompt 模板** — 智能体不是代码中的类，而是技能目录中的 `.md` 模板文件
2. **Controller 即编排器** — 主会话不是"另一个智能体"，而是唯一的协调者，拥有全局上下文
3. **子智能体无状态** — 每次派遣都是全新实例，不继承会话历史，避免上下文污染
4. **技能定义角色** — 同一个 AI 模型在不同技能指导下扮演完全不同的角色
5. **审阅者与实现者对立** — spec-reviewer 的核心指令是"不要信任实现者的报告"
