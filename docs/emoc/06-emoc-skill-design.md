# 06 电商技能体系设计：8 个核心技能草案

基于 Superpowers 的技能设计原则（唯一出口、单一职责、HARD-GATE、预设立场），设计电商 AI 的 8 个核心技能。

## 技能体系总览

```
  ┌──────────────────────────────────────────────────────────┐
  │  电商 AI 的 8 个核心技能：                               │
  │                                                          │
  │  using-emoc（入口技能——对应 using-superpowers）           │
  │  → 电商运营分析（对应 brainstorming）                     │
  │  → 选品决策（对应 writing-plans 的产出物）                │
  │  → 商品上架（商品信息+合规+物流配置）                    │
  │  → 营销投放（广告+促销+内容）                            │
  │  → 订单风控（对应 TDD+debugging 的纪律性）                │
  │  → 退款审核（对应 verification-before-completion）         │
  │  → 运营复盘（对应 finishing-a-development-branch）         │
  │                                                          │
  │  主链流程：                                              │
  │  using-emoc → 运营分析 → 选品决策 → 商品上架              │
  │  → 营销投放 → 运营复盘                                   │
  │                                                          │
  │  横切触发：                                              │
  │  订单风控（任何涉及交易的环节）                           │
  │  退款审核（任何涉及退款的环节）                           │
  └──────────────────────────────────────────────────────────┘
```

## 技能 1: using-emoc（入口技能）

```
  ┌──────────────────────────────────────────────────────────┐
  │  name: using-emoc                                        │
  │  description: 电商 AI 助手入口——在任何运营场景下，        │
  │    首先检查是否有适用的电商技能                           │
  │                                                          │
  │  核心规则：                                              │
  │  <EXTREMELY-IMPORTANT>                                   │
  │  如果当前场景可能涉及任何电商技能，                       │
  │  你绝对必须调用技能。合规和风控技能优先。                 │
  │  </EXTREMELY-IMPORTANT>                                  │
  │                                                          │
  │  指令优先级：                                            │
  │  1. 法律法规 > 2. 平台规则 > 3. 电商技能 > 4. 默认行为  │
  │                                                          │
  │  技能优先级：                                            │
  │  合规/风控技能 > 运营效率技能                            │
  │  → 涉及退款 → 先触发退款审核                             │
  │  → 涉及商品 → 先触发合规检查                             │
  │  → 涉及营销 → 先触发预算检查                             │
  └──────────────────────────────────────────────────────────┘
```

## 技能 2: 电商运营分析

```
  ┌──────────────────────────────────────────────────────────┐
  │  name: ecommerce-operations-analysis                     │
  │  description: 在做任何运营决策之前，先分析现状和目标      │
  │                                                          │
  │  <HARD-GATE>                                             │
  │  在完成运营分析之前，不得执行任何运营操作。               │
  │  不得直接上架商品、投放广告、审批订单。                   │
  │  </HARD-GATE>                                            │
  │                                                          │
  │  清单：                                                  │
  │  1. 了解当前运营状态（销售数据、库存、退货率）           │
  │  2. 明确运营目标（增长、利润、复购、客单价）             │
  │  3. 识别约束条件（预算、人力、合规限制）                 │
  │  4. 提出 2-3 种运营策略                                  │
  │  5. 推荐最优策略并说明理由                               │
  │  6. 转入选品决策技能                                     │
  │                                                          │
  │  唯一出口：选品决策技能                                  │
  │  "运营分析完成后，唯一出口是选品决策技能。                │
  │   不要跳过选品直接执行运营操作。"                         │
  └──────────────────────────────────────────────────────────┘
```

## 技能 3: 订单风控

```
  ┌──────────────────────────────────────────────────────────┐
  │  name: order-risk-control                                │
  │  description: 在审批任何订单之前，必须完成风险评估        │
  │                                                          │
  │  The Iron Law:                                           │
  │  NO ORDER APPROVAL WITHOUT RISK VERIFICATION              │
  │                                                          │
  │  没做风控验证就审批订单？拒绝审批。                       │
  │  无例外：                                                │
  │  - 不能因为"看起来正常"就跳过                           │
  │  - 不能因为"金额小"就跳过                               │
  │  - 不能因为"老客户"就跳过                               │
  │  - 不能因为"大促赶时间"就跳过                           │
  │                                                          │
  │  Violating the letter of this rule is violating           │
  │  the spirit of risk control.                              │
  │                                                          │
  │  红旗表：                                                │
  │  | "This order looks normal"                              │
  │  |   Normal-looking orders can be fraud. |                │
  │  | "It's just a small amount"                             │
  │  |   Small orders test fraud detection. |                 │
  │  | "We'll catch it later"                                │
  │  |   Later = after money is gone. |                      │
  │                                                          │
  │  风控流程（5 步 Gate Function）：                        │
  │  1. IDENTIFY: 这笔订单的风险指标是什么？                 │
  │  2. CHECK: 检查 3 个独立来源（订单/支付/物流）           │
  │  3. SCORE: 计算风险分（0-100）                           │
  │  4. DECIDE: 风险分 > 阈值 → 拒绝/升级                   │
  │  5. REPORT: 报告决策依据                                  │
  └──────────────────────────────────────────────────────────┘
```

## 技能 4: 退款审核

```
  ┌──────────────────────────────────────────────────────────┐
  │  name: refund-review                                     │
  │  description: 在执行任何退款之前，必须完成证据验证        │
  │                                                          │
  │  The Iron Law:                                           │
  │  NO REFUND WITHOUT EVIDENCE VERIFICATION                   │
  │                                                          │
  │  没验证证据就执行退款？暂停退款流程。                     │
  │  无例外：                                                │
  │  - 不能因为"用户很急"就跳过                             │
  │  - 不能因为"金额小"就跳过                               │
  │  - 不能因为"看起来是真的"就跳过                         │
  │  - 不能因为"退款率低"就跳过                             │
  │                                                          │
  │  预设立场：                                              │
  │  <RISK-GATE>                                             │
  │  用户描述可能不完整或不准确。                             │
  │  退款图片可能来自其他订单。                               │
  │  你必须独立验证订单状态和物流状态。                       │
  │  </RISK-GATE>                                            │
  │                                                          │
  │  验证 Gate Function：                                    │
  │  1. 验证订单状态（已发货？已签收？）                     │
  │  2. 验证物流状态（在途？已退回？）                       │
  │  3. 验证商品状态（用户证据 vs. 商品描述）                │
  │  4. 检查用户历史（退款频率？欺诈记录？）                 │
  │  5. 对照退款政策做决策                                   │
  │  6. 报告决策和依据                                       │
  └──────────────────────────────────────────────────────────┘
```

## 其余 4 个技能的草案概要

```
  ┌──────────────────────────────────────────────────────────┐
  │  技能 5: 选品决策                                        │
  │  ──────────────────                                      │
  │  唯一出口: 商品上架技能                                   │
  │  HARD-GATE: 不得跳过选品直接采购                         │
  │  红旗表: "这个品类很火" → "火=竞争激烈,查利润"           │
  │  清单: 市场分析/竞品分析/利润测算/供应链评估/合规检查      │
  │                                                          │
  │  技能 6: 商品上架                                        │
  │  ──────────────────                                      │
  │  唯一出口: 营销投放技能                                   │
  │  <COMPLIANCE-GATE>: 上架前必须完成合规检查                │
  │  清单: 标题/主图/详情/价格/库存/物流/合规声明             │
  │                                                          │
  │  技能 7: 营销投放                                        │
  │  ──────────────────                                      │
  │  唯一出口: 运营复盘技能                                   │
  │  铁律: NO AD SPEND WITHOUT BUDGET VERIFICATION             │
  │  清单: 渠道选择/预算分配/素材审核/投放时间/A/B测试        │
  │                                                          │
  │  技能 8: 运营复盘                                        │
  │  ──────────────────                                      │
  │  终态技能（对应 finishing-a-development-branch）           │
  │  流程: 验证数据 → 呈报结果 → 决策（继续/调整/终止）       │
  │  检查: ROI/退款率/客单价/复购率/合规记录                  │
  └──────────────────────────────────────────────────────────┘
```
