# Superpowers 最佳实践向电商领域的平移可行性分析

Superpowers 是一套 AI 工程化框架——用协议、约束、注意力工程和裁量空间控制来让 AI 可靠地完成复杂工作流。这些实践不限于软件开发——它们是**人机协作的通用方法论**。本文分析哪些实践可以平移到电商领域，以及如何平移。

## 核心判断

```
  ┌──────────────────────────────────────────────────────────┐
  │  Superpowers 的实践分为三类：                            │
  │                                                          │
  │  A. 通用方法论（可直接平移）                              │
  │     → 协议设计、注意力工程、裁量空间控制                  │
  │     → 不依赖软件开发上下文                                │
  │                                                          │
  │  B. 领域适配型（需要重新设计）                            │
  │     → 技能体系、子智能体流程、审阅机制                    │
  │     → 框架可复用，但内容需电商化                         │
  │                                                          │
  │  C. 不可平移（依赖开发上下文）                            │
  │     → TDD、git worktree、代码审阅                        │
  │     → 深度耦合于软件工程实践                              │
  │                                                          │
  │  本系列重点分析 A 类和 B 类。                             │
  └──────────────────────────────────────────────────────────┘
```
