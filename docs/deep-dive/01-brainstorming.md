# 阶段 ①：头脑风暴（brainstorming）

所有创造性工作的**强制入口**。无论项目多简单都必须经过此阶段。

> "简单项目是不经审视的假设浪费最多工作的地方。" —— brainstorming skill

## 流程

```
  ┌──────────────┐
  │ 探索项目上下文 │  读文件、文档、最近提交
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 提供视觉伴侣  │  如果涉及视觉问题，单独一条消息询问
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 逐一提问澄清  │  每次只问一个问题，尽量多选题
  └──────┬───────┘  如果是多个独立子系统 → 先分解！
         ▼
  ┌──────────────┐
  │ 提出 2-3 方案 │  含权衡和推荐
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 分段呈现设计  │  按架构/组件/数据流/错误处理/测试分段
  └──────┬───────┘  每段确认后才继续
         ▼
  ┌──────────────┐
  │ 写设计文档    │  docs/superpowers/specs/YYYY-MM-DD-<topic>-design.md
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 规范自审      │  查占位符/矛盾/歧义/范围
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 用户审阅规范  │  用户批准后才继续
  └──────┬───────┘
         ▼
  ┌──────────────┐
  │ 转入写计划    │  调用 writing-plans skill（唯一出口）
  └──────────────┘
```

## 硬门禁

在用户批准设计之前，**绝不**：
- 调用任何实现技能
- 写任何代码
- 调用 `frontend-design`、`mcp-builder` 等实现类技能

唯一允许的后续技能是 `writing-plans`。

## 范围分解

如果请求描述了多个独立子系统（如"建一个含聊天/存储/计费/分析的平台"），**先分解成子项目**：

```
  "建一个平台"（太大）
       │
       ▼ 分解
  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐
  │  聊天   │  │  存储   │  │  计费   │  │  分析   │
  │ 子项目  │  │ 子项目  │  │ 子项目  │  │ 子项目  │
  └────┬────┘  └─────────┘  └─────────┘  └─────────┘
       │
       ▼ 每个独立走
  spec → plan → implement 循环
```

## 设计原则

- **一个一个问题** — 不要一次问多个问题
- **多选题优先** — 比开放式问题更容易回答
- **YAGNI 无情** — 从所有设计中删除不需要的功能
- **探索替代方案** — 在确定前总是提出 2-3 种方法
- **增量验证** — 呈现设计，获得批准后再继续
- **为隔离而设计** — 每个单元有一个清晰目的、定义良好的接口，可独立理解和测试

## 规范自审清单

写完设计文档后，用新鲜眼光检查：

| 检查项 | 说明 |
|--------|------|
| 占位符扫描 | 搜索 "TBD"、"TODO"、不完整章节、模糊需求 |
| 内部一致性 | 各章节是否矛盾？架构是否匹配功能描述？ |
| 范围检查 | 是否足够聚焦于单个实施计划？是否需要分解？ |
| 歧义检查 | 任何需求是否可以被两种方式解读？ |

发现问题就即时修复，不需要重新审阅。

## 用户审阅门

自审通过后，必须请用户审阅：

> "规范已写入并提交到 `<path>`。请审阅并告知在开始编写实施计划前是否需要修改。"

用户批准后才调用 `writing-plans`。

## 视觉伴侣

当即将到来的问题涉及视觉内容（模型图、布局、图表）时，提供浏览器伴侣选项。**此提议必须是独立消息**，不与澄清问题混合。

| 内容类型 | 使用方式 |
|---------|---------|
| 模型图、线框图、布局对比、架构图 | 浏览器（视觉） |
| 需求问题、概念选择、权衡列表 | 终端（文本） |
