# 06 技能内容排序：先约束后自由的认知设计

Superpowers 的技能文件不是随意排列的——它们的顺序遵循"先约束后自由"的认知设计原则。AI 读到的前几行决定了它如何理解后续内容。

## 四种技能的排序模式

```
  ┌──────────────────────────────────────────────────────────┐
  │  模式 A: 纪律型技能（TDD, debugging, verification）      │
  │  ──────────────────────────────────────────────          │
  │  1. "Violating the letter is violating the spirit"        │
  │  2. Iron Law（全大写 + 代码块）                          │
  │  3. 详细流程                                             │
  │  4. 理性化反驳表                                        │
  │  5. Red Flags - STOP                                     │
  │  6. Final Rule（重述 Iron Law）                           │
  │  → 首尾是约束，中间是指导                               │
  │  → 约束"夹住"了指导                                    │
  │                                                          │
  │  模式 B: 设计型技能（brainstorming）                      │
  │  ──────────────────────────────────────────────          │
  │  1. HARD-GATE（绝对禁止）                                │
  │  2. Anti-Pattern（最常见绕过方式）                       │
  │  3. Checklist（9 步流程）                                │
  │  4. Process Flow（流程图）                                │
  │  5. Detailed Process（细节指导）                          │
  │  6. Key Principles（紧凑原则）                            │
  │  7. Implementation（唯一出口重述）                        │
  │  → 开头是禁止，中间是流程，末尾是出口                   │
  │  → 禁止 + 流程 + 出口 = 注意力的完整路径                │
  │                                                          │
  │  模式 C: 协调型技能（subagent-driven, writing-plans）     │
  │  ──────────────────────────────────────────────          │
  │  1. Overview（角色设定）                                 │
  │  2. When to Use（决策图）                                │
  │  3. The Process（流程图）                                │
  │  4. Detailed guidance                                    │
  │  5. Example Workflow（具体例子）                          │
  │  6. Red Flags（Never 列表）                               │
  │  7. Integration（技能依赖声明）                           │
  │  → 开头是角色，中间是流程+例子，末尾是护栏+依赖        │
  │  → 角色设定比约束更柔和但仍然锚定注意力                 │
  │                                                          │
  │  模式 D: 入口型技能（using-superpowers）                  │
  │  ──────────────────────────────────────────────          │
  │  1. EXTREMELY-IMPORTANT（最高强制）                       │
  │  2. Instruction Priority（优先级层次）                    │
  │  3. The Rule + Flow Diagram（匹配规则+流程图）            │
  │  4. Red Flags（12 条理性化）                              │
  │  5. Skill Priority / Types / User Instructions            │
  │  → 全篇都是约束和规则——没有"自由"内容                 │
  │  → 这是"纯约束"型技能——因为它的唯一目的就是确保       │
  │    其他技能被正确调用                                   │
  └──────────────────────────────────────────────────────────┘
```

## 排序的认知心理学基础

```
  ┌──────────────────────────────────────────────────────────┐
  │  首因效应 (Primacy Effect)                               │
  │  ──────────────────────────                              │
  │  → AI 读到的前几行内容获得更高的初始权重                 │
  │  → 前几行定义了"这个技能是关于什么的"                  │
  │  → 如果前几行是约束 → AI 理解"这个技能要求我不做X"     │
  │  → 如果前几行是指导 → AI 理解"这个技能让我做X"         │
  │  → 约束在前 = AI 先建立"禁止"框架，再在框架内行动       │
  │  → 指导在前 = AI 先建立"行动"框架，约束可能被忽略       │
  │                                                          │
  │  近因效应 (Recency Effect)                               │
  │  ──────────────────────────                              │
  │  → AI 最后读到的内容在后续行为中权重更高                 │
  │  → 如果最后是 Final Rule → AI 带着约束离开技能          │
  │  → 如果最后是细节 → AI 带着细节离开，可能忽略约束       │
  │  → 约束在尾 = AI 离开时最强烈的记忆是"不能做X"         │
  │                                                          │
  │  框架效应 (Framing Effect)                               │
  │  ──────────────────────────                              │
  │  → 同一内容的不同呈现顺序影响 AI 的理解                  │
  │  → "HARD-GATE: 不要写代码。现在做设计。"                 │
  │     vs. "现在做设计。HARD-GATE: 不要写代码。"            │
  │  → 前者：AI 先建立"禁止"框架，然后在框架内设计         │
  │  → 后者：AI 先建立"设计"框架，禁止是附加的             │
  │  → 前者的约束力更强——因为框架已经设定                  │
  │                                                          │
  │  Superpowers 的排序策略：                                │
  │  约束 → 流程 → 细节 → 约束（重述）                     │
  │  = 首因效应（约束）+ 近因效应（重述约束）               │
  │  = 框架效应（先"不能做什么"再"能做什么"）              │
  │  = 三种认知效应的协同利用                               │
  └──────────────────────────────────────────────────────────┘
```
