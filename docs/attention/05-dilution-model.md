# 05 注意力稀释模型：量化与预测

## 稀释的量化模型

```
  ┌──────────────────────────────────────────────────────────┐
  │  定义：                                                  │
  │  A(i,t) = 指令 i 在时刻 t 的注意力权重                  │
  │                                                          │
  │  影响因素：                                              │
  │  1. 位置距离 d(i,t) = 指令 i 到当前位置的 token 距离     │
  │  2. 信号强度 s(i) = 指令 i 的结构性权重                  │
  │  3. 竞争数量 N(t) = 时刻 t 上下文中的指令总数            │
  │  4. 稀释比例 r(t) = 非指令内容占总上下文的比例           │
  │                                                          │
  │  简化模型：                                              │
  │  A(i,t) = s(i) / (d(i,t)^a * N(t)^b * (1 + r(t)))      │
  │                                                          │
  │  其中 a, b 是衰减指数（经验值约 0.3-0.5）               │
  │                                                          │
  │  模型预测：                                              │
  │  1. 位置越远，权重越低（距离衰减）                      │
  │  2. 信号越强，权重越高（结构性增强）                    │
  │  3. 竞争越多，每条权重越低（竞争衰减）                  │
  │  4. 稀释比例越高，权重越低（稀释衰减）                  │
  └──────────────────────────────────────────────────────────┘
```

## 信号强度的分级

```
  ┌──────────────────────────────────────────────────────────┐
  │  信号强度 s(i) 的分级（1-10）：                         │
  │                                                          │
  │  Level 10: 铁律 + XML + 全大写 + 代码块                  │
  │  → 例：EXTREMELY-IMPORTANT + ABSOLUTELY MUST             │
  │  → 实际效果：接近 100% 服从                              │
  │                                                          │
  │  Level 9: HARD-GATE XML 标签                             │
  │  → 例：<HARD-GATE> Do NOT... </HARD-GATE>                │
  │  → 实际效果：~95% 服从                                   │
  │                                                          │
  │  Level 8: 铁律全大写（无 XML 标签）                      │
  │  → 例：NO PRODUCTION CODE WITHOUT A FAILING TEST          │
  │  → 实际效果：~90% 服从                                   │
  │                                                          │
  │  Level 7: 红旗表 + "STOP"                                │
  │  → 例："Red Flags - STOP and Start Over"                  │
  │  → 实际效果：~85% 服从                                   │
  │                                                          │
  │  Level 6: 宣布仪式                                       │
  │  → 例："I'm using the writing-plans skill..."             │
  │  → 实际效果：~80% 服从                                   │
  │                                                          │
  │  Level 5: 流程图路径                                     │
  │  → 例：dot 图中的决策节点和终态                         │
  │  → 实际效果：~75% 服从                                   │
  │                                                          │
  │  Level 4: 清单项                                         │
  │  → 例："You MUST create a task for each of these"         │
  │  → 实际效果：~70% 服从                                   │
  │                                                          │
  │  Level 3: 理性化反驳表                                   │
  │  → 例："Excuse | Reality" 表格                           │
  │  → 实际效果：~60% 服从（依赖 AI 自检）                  │
  │                                                          │
  │  Level 2: 普通规则                                       │
  │  → 例："Follow existing patterns"                         │
  │  → 实际效果：~50% 服从                                   │
  │                                                          │
  │  Level 1: 建议/偏好                                      │
  │  → 例："Prefer multiple choice when possible"             │
  │  → 实际效果：~30% 服从                                   │
  │                                                          │
  │  注意：这些百分比是经验估计，不是精确测量               │
  │  → 不同模型、不同上下文的实际服从率不同                 │
  │  → 但相对排序是稳定的：Level 10 > Level 1               │
  │  → 信号强度的分级 = 注意力工程的设计空间                │
  └──────────────────────────────────────────────────────────┘
```

## 稀释的仿真：一个完整功能开发会话

```
  ┌──────────────────────────────────────────────────────────┐
  │  假设：一个 5 任务的功能开发，每个任务 3 轮审阅          │
  │  总计约 25 轮对话                                        │
  │                                                          │
  │  时刻 T0: using-superpowers 注入                         │
  │  A(1%, T0) = 10 / (1 * 1 * 1) = 10 (满权重)            │
  │                                                          │
  │  时刻 T5: brainstorming 完成                             │
  │  A(1%, T5) = 10 / (5^0.3 * 3 * 1.5) ≈ 2.8              │
  │  A(HARD-GATE, T5) = 9 / (3^0.3 * 3 * 1.5) ≈ 2.9        │
  │  → 两者权重相当——但 HARD-GATE 不再需要了               │
  │                                                          │
  │  时刻 T15: subagent-driven 执行中                        │
  │  A(1%, T15) = 10 / (15^0.3 * 8 * 3) ≈ 0.6              │
  │  → 1% 原则的权重降到 0.6——接近注意力阈值               │
  │  → AI 可能开始跳过技能检查                              │
  │  → 这就是 using-superpowers 最脆弱的时刻                │
  │                                                          │
  │  时刻 T25: finishing 阶段                                │
  │  A(1%, T25) = 10 / (25^0.3 * 12 * 5) ≈ 0.3              │
  │  → 1% 原则几乎不可见                                    │
  │  → 如果此时出现意外（如 bug），AI 可能不触发 debugging  │
  │  → 需要横切技能的 description 自带信号增强              │
  │                                                          │
  │  缓解：横切技能的信号增强                                │
  │  debugging 的 description:                               │
  │  "Use when encountering ANY bug..."                       │
  │  → "ANY" = 全大写信号增强                               │
  │  → 加上技能加载后的铁律                                 │
  │  → s(debugging) ≈ 9                                      │
  │  → A(debugging, T25) = 9 / (1 * 12 * 5) ≈ 0.15          │
  │  → 虽然低，但 debugging 触发条件（bug/失败）            │
  │    本身就是强信号——上下文中出现了"错误消息"             │
  │  → 错误消息的 s ≈ 8（代码输出的信号强度高）            │
  │  → 两者叠加可能足够触发                                 │
  │                                                          │
  │  关键洞察：                                              │
  │  → 注意力稀释是不可避免的                                │
  │  → 但可以通过信号增强和冗余来补偿                       │
  │  → 最危险的时刻 = 长对话 + 旧规则 + 新问题              │
  │  → 最安全的策略 = 子智能体隔离（避免累积）              │
  └──────────────────────────────────────────────────────────┘
```
